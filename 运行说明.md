# SocialMediaCut 运行指南

## 项目简介

SocialMediaCut 是一个功能强大的直播录制工具,支持多个国内直播平台(抖音、B站、斗鱼、虎牙、快手)的直播监控和自动录制。

---

## 一、环境准备

### 1.1 Python 环境

- **Python 版本**: Python 3.11 或更高版本
- **检查方法**:
  ```bash
  python --version
  ```

### 1.2 安装依赖

```bash
pip install -r requirements.txt
```

**依赖列表**:
- `requests` - HTTP 请求库
- `PyYAML` - YAML 配置文件解析
- `ttkbootstrap` - 美化的 Tkinter UI 组件

## 三、配置说明

### 3.1 配置文件

主配置文件: `config.yaml`

```yaml
# API 服务配置
api:
  enable: true          # 是否启用 API 服务
  host: 0.0.0.0       # 监听地址
  port: 8888          # 监听端口

# 弹幕配置
danmaku:
  enable: false        # 是否启用弹幕录制
  output_dir: recordings/danmaku

# 日志配置
log:
  console: true        # 是否输出到控制台
  file: logs/app.log  # 日志文件路径
  level: info         # 日志级别 (debug/info/warning/error)

# 监控配置
monitor:
  interval_sec: 60    # 监控间隔(秒)
  max_workers: 8      # 最大并发数
  request_timeout_sec: 10  # 请求超时时间

# 录制配置
record:
  filename_template: '{platform}/{room_id}/{year}/{month}/{day}/{title}_{time}'
  format: flv         # 录制格式
  output_dir: recordings  # 录制输出目录
  quality: origin     # 录制质量 (origin/high/medium/low)

# 监控房间列表
rooms:
  - platform: douyin  # 平台 (douyin/bilibili/douyu/huya/kuaishou)
    room_id: '123456'      # 房间ID
    name: '测试房间'         # 房间名称(可选)
    auto_record: true       # 是否自动录制

# Webhook 配置
webhook:
  enable: false        # 是否启用 Webhook
  events:
    - live_start      # 开播事件
    - live_end        # 下播事件
    - download_complete  # 下载完成事件
  url: ''            # Webhook URL
```

## 五、API 接口

当 `api.enable: true` 时,程序会启动 HTTP API 服务。

### 基础信息

- **地址**: `http://localhost:8888`
- **格式**: JSON

### 主要接口

#### 1. 获取所有房间
```http
GET /api/rooms
```

响应:
```json
{
  "success": true,
  "data": [
    {
      "platform": "douyin",
      "room_id": "123456",
      "name": "测试房间",
      "is_live": true,
      "is_recording": true
    }
  ]
}
```

#### 2. 添加房间
```http
POST /api/rooms
Content-Type: application/json

{
  "platform": "douyin",
  "room_id": "123456",
  "name": "测试房间",
  "auto_record": true
}
```

#### 3. 删除房间
```http
DELETE /api/room/{platform}/{room_id}
```

#### 4. 开始录制
```http
POST /api/recording/start
Content-Type: application/json

{
  "platform": "douyin",
  "room_id": "123456"
}
```

#### 5. 停止录制
```http
POST /api/recording/stop
Content-Type: application/json

{
  "platform": "douyin",
  "room_id": "123456"
}
```

#### 6. 搜索房间
```http
GET /api/targets/search?platform=douyin&query=关键词
```

#### 7. 获取状态
```http
GET /api/status
```

#### 8. 获取支持的平台
```http
GET /api/platforms
```

## 六、常见问题

### Q1: GUI 启动时报错 "ModuleNotFoundError: No module named 'ttkbootstrap'"

**原因**: 缺少 ttkbootstrap 依赖

**解决**:
```bash
pip install ttkbootstrap
```

### Q2: 录制文件保存在哪里?

**答案**: 默认保存在 `recordings/` 目录下,可以通过修改 `config.yaml` 中的 `record.output_dir` 来更改。

### Q3: 如何支持更多平台?

**答案**: 目前支持抖音、B站、斗鱼、虎牙、快手。要添加新平台,需要在 `platforms/` 目录下创建新的解析器。

### Q4: 直播间一直显示"未开播"

**原因**:
1. 房间ID 错误
2. 直播间已被封禁
3. 网络问题

**解决**:
1. 检查房间ID是否正确
2. 尝试在浏览器中打开直播间确认
3. 检查网络连接

### Q5: 录制中断或失败

**原因**:
1. 网络不稳定
2. 存储空间不足
3. 磁盘权限问题

**解决**:
1. 检查网络连接
2. 检查磁盘空间
3. 检查 `recordings/` 目录的写入权限

### Q6: 如何修改监控频率?

**答案**: 修改 `config.yaml` 中的 `monitor.interval_sec` 参数,单位为秒。

## 七、高级功能

### 7.1 自动录制

在 `config.yaml` 的 `rooms` 配置中,设置 `auto_record: true`,当直播间开播时会自动开始录制。

### 7.2 Webhook 通知

配置 Webhook 后,可以在特定事件发生时接收通知:

```yaml
webhook:
  enable: true
  events:
    - live_start
    - live_end
    - download_complete
  url: 'http://your-server.com/webhook'
```

### 7.3 日志级别

修改 `log.level` 可以调整日志详细程度:
- `debug` - 最详细的日志
- `info` - 默认级别
- `warning` - 只记录警告和错误
- `error` - 只记录错误

## 九、性能优化建议

1. **监控频率**: 根据需求调整 `monitor.interval_sec`,不要太频繁以免被平台限制
2. **并发数**: `monitor.max_workers` 设置为 4-8 之间,过高可能导致问题
3. **录制质量**: `record.quality` 设置为 `origin` 获取最高质量,但需要更高带宽
4. **存储空间**: 定期清理旧的录制文件,避免磁盘空间不足
